# Learning paths

A learning path is a suggested order of projects for a particular goal. Pick the
path that matches what you want to build, then work through its projects in
order.

Projects marked _(planned)_ do not exist yet; they show the intended direction.
See the [roadmap](roadmap.md) for status, and the level pages
([beginner](../beginner/README.md), [intermediate](../intermediate/README.md),
[advanced](../advanced/README.md)) for what is available now.

## Difficulty levels

| Level | Who it is for | Typical scope |
|---|---|---|
| Beginner | Knows basic Python syntax | A small, single-purpose script, standard library only |
| Intermediate | Comfortable with functions, classes, and packages | A structured application with storage or multiple interfaces |
| Advanced | Comfortable structuring code into layers | A service, pipeline, or integration with external systems |

## Path 1: Python fundamentals

Build confidence with the language and good habits (structure, tests, clear
CLIs).

1. [Expense Tracker CLI](../beginner/01-expense-tracker/README.md) _(beginner)_
2. [Safe File Organizer CLI](../beginner/02-file-organizer/README.md) _(beginner)_

## Path 2: Automation and tooling

Learn to automate real tasks safely, with dry-run previews and clear output.

1. [Safe File Organizer CLI](../beginner/02-file-organizer/README.md) _(beginner)_
2. Additional automation utilities _(planned)_

## Path 3: Backend and APIs

Move from scripts to services with clear boundaries and tested endpoints.

1. [URL Shortener API](../intermediate/01-url-shortener-api/README.md) _(intermediate)_
2. Additional backend services _(planned)_

## Path 4: AI-assisted applications

Apply Python to document processing and AI-assisted workflows with an emphasis
on safety and privacy.

1. Document processing utilities _(planned)_
2. AI document assistant _(planned)_
