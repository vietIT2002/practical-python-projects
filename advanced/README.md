# Advanced projects

Larger projects for people who want to practise production-minded engineering:
services, pipelines, and integrations with clear boundaries.

## What "advanced" means here

- Involves a service, a data pipeline, or external integrations.
- Separates domain logic from input/output details (see the
  [architecture decisions](../docs/architecture/)).
- Handles configuration, errors, and security deliberately.
- Includes integration tests for important boundaries, not only unit tests.

**You should be comfortable with:** structuring a codebase into layers, testing
external boundaries with fakes, and reasoning about failure and security.

self-contained, with its own dependencies and lockfile. See the
[repository map](../docs/repository-map.md) for naming rules and project status
definitions.
