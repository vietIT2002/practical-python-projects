# Governance

This document describes how decisions are made in this project. It is
intentionally lightweight and will grow if the community grows.

## Roles

- **Maintainer** — currently [@vietIT2002](https://github.com/vietIT2002).
  Reviews contributions, sets direction, and is responsible for releases and
  security handling.
- **Contributors** — anyone who opens issues or pull requests.

## How decisions are made

- Routine changes (fixes, docs, new projects that follow the conventions) are
  decided through normal pull-request review by a maintainer.
- Larger changes (new conventions, architecture changes, tooling changes) start
  as an issue or discussion so the reasoning can be recorded, and significant
  outcomes are captured as an
  [architecture decision record](docs/architecture/README.md).

## Adding maintainers

As the project grows, contributors with a sustained track record of high-quality
contributions may be invited to become maintainers. There is no fixed quota; the
goal is reliable, respectful stewardship.

## Code of Conduct

All participation is governed by the [Code of Conduct](CODE_OF_CONDUCT.md).
