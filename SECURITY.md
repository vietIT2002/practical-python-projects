# Security policy

We take the security of this project seriously, including the example code,
because learners may adapt it in their own work.

## Supported versions

This repository is pre-1.0 and evolving. Security fixes are applied to the
latest state of the `main` branch. Once tagged releases begin, this section will
list the versions that receive security updates.

| Version | Supported |
|---|---|
| `main` (latest) | Yes |

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues,
discussions, or pull requests.**

Report privately using GitHub's built-in private vulnerability reporting:

1. Go to the repository's **Security** tab.
2. Select **Report a vulnerability**.
3. Describe the issue, the affected file or project, and steps to reproduce it.

If you are unable to use that channel, open a minimal public issue that says only
that you have a security report and asks for a private contact route — do not
include any details.

## What to include

- A clear description of the problem and its impact.
- The affected project or file.
- Steps to reproduce, ideally with a minimal example.
- Any suggested remediation, if you have one.

## What to expect

- We aim to acknowledge a report within a few days.
- We will confirm the issue, discuss a fix, and agree on disclosure timing with
  you.
- We will credit reporters who wish to be named once a fix is available.

We do not offer a paid bug bounty for this project.
